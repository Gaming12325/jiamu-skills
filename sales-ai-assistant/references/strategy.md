# 销售策略与规划场景库

本文件包含销售策略类别下的扩展场景（除SKILL.md中的1个高频场景外）。

---

## 场景1：设计区域规划框架

**触发关键词**：区域规划、地区分配、territory planning、销售区域、覆盖范围

**必填变量**：
- `[团队人数]`：销售团队规模
- `[目标行业]`：重点关注的行业领域
- `[地区范围]`：覆盖的地理区域

**选填变量**：
- `[历史收入数据]`：各区域的历史业绩
- `[目标设定]`：下一财年的收入目标

**提示词模板**：
```
为下一财年制定区域规划指南。

团队信息：
- 团队人数：[团队人数]
- 目标行业：[目标行业]
- 地区范围：[地区范围]
- 历史收入：[历史收入数据]（如有）
- 收入目标：[目标设定]（如有）

输出要求：
制定一份区域规划框架，包含：
1. 区域划分建议（基于地理、行业或客户规模）
2. 资源分配方法（如何分配销售代表）
3. 覆盖范围示例方案（每个代表负责的区域/账户）
4. 平衡考虑因素（机会公平性、旅行成本、市场潜力）
5. 实施时间表和里程碑

格式：结构化的规划文档，包含示例分配表格
```

**使用示例**：
- 用户："我们有10个销售，要覆盖华东、华南、华北三个区域，主要做SaaS"
- Skill生成区域规划框架

## 场景2：利用公司概况数据确定账户优先级

**触发关键词**：账户优先级、客户排序、account prioritization、潜在客户筛选

**必填变量**：
- `[账户列表]`：待排序的客户/潜在客户列表
- `[优先级标准]`：排序依据（行业、规模、融资、技术栈等）

**选填变量**：
- `[现有客户数据]`：已有客户的成功案例特征

**提示词模板**：
```
我有以下账户列表需要确定优先级。

账户列表：
[账户列表]

优先级标准：
[优先级标准]
示例标准：行业匹配度、公司规模、融资阶段、技术栈、增长速度、决策周期等

参考数据（如有）：
[现有客户数据]

输出要求：
1. 根据标准对账户进行评分和排序
2. 输出排名列表（Top 10-20）
3. 为每个账户提供排序理由（1-2句话）
4. 标注高优先级账户的共同特征
5. 建议每个优先级层级的行动策略

格式：排名表格 + 特征分析 + 行动建议
```

**使用示例**：
- 用户："我有100个潜在客户，想知道应该先联系哪些，我们的理想客户是50-500人的科技公司"
- Skill根据标准进行账户排序

## 场景3：利用加权评分发现高潜力账户

**触发关键词**：加权评分、lead scoring、潜力评估、客户评分

**必填变量**：
- `[账户数据]`：包含客户信息的列表或数据导出
- `[评分规则]`：各项指标的权重（如：公司规模30%，互动评分40%，意向信号30%）

**选填变量**：
- `[评分阈值]`：定义高/中/低潜力的分数线

**提示词模板**：
```
根据加权评分规则对账户进行评分，识别高潜力客户。

账户数据：
[账户数据]

评分规则：
[评分规则]
示例：公司规模（30%）+ 网站互动评分（40%）+ 意向信号（30%）

评分阈值（如有）：
[评分阈值]
示例：≥80分为高潜力，60-79为中等，<60为低优先级

输出要求：
1. 为每个账户计算综合得分
2. 输出排名前10的高潜力账户
3. 为每个账户附上得分明细和解释
4. 可视化得分分布（如：直方图描述）
5. 建议针对不同潜力层级的跟进策略

格式：得分排名表 + 评分解释 + 行动建议
```

**使用示例**：
- 用户："帮我评估这些leads，按照公司规模、访问次数、下载资料数量来打分"
- Skill进行加权评分并输出排名

## 场景4：区域市场进入规划

**触发关键词**：市场进入、新市场、market entry、区域扩张、国际化

**必填变量**：
- `[产品/解决方案]`：要推广的产品或服务
- `[目标市场]`：地区、国家或细分市场

**选填变量**：
- `[现有市场经验]`：在其他市场的成功经验
- `[资源限制]`：预算、团队规模等限制

**提示词模板**：
```
评估我们的[产品/解决方案]进入[目标市场]的可行性。

产品信息：
[产品/解决方案]

目标市场：
[目标市场]

现有经验（如有）：
[现有市场经验]

资源限制（如有）：
[资源限制]

分析要求：
请研究以下方面并形成市场进入准备情况评估：
1. 当地购买行为和决策流程
2. 竞争格局（主要竞争对手、市场份额）
3. 经济和商业环境（市场规模、增长潜力）
4. 监管和合规问题
5. 文化和语言考虑因素
6. 进入壁垒和成功关键因素

输出格式：
- 市场分析报告（结构化）
- 可行/不可行结论
- 附上参考文献和数据来源
- 具体的行动步骤建议（如果可行）
```

**使用示例**：
- 用户："我们想把CRM产品推到东南亚市场，不知道可行性如何"
- Skill进行市场研究并给出评估报告

## 使用说明

这些策略场景通常需要更深入的分析和规划，适合中高层销售管理者使用。

**关键特点**：
- 更注重战略和规划，而非执行层面的沟通
- 通常需要数据支撑和分析
- 输出内容更结构化、更正式

**优先级**：
当用户描述涉及"规划"、"策略"、"分析市场"等关键词时，优先匹配这些场景。
