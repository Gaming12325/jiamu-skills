# 竞争情报与赋能场景库

本文件包含竞争情报类别下的扩展场景（除SKILL.md中的2个高频场景外）。

---

## 场景1：竞争定位分析

**触发关键词**：竞争分析、竞品研究、competitor analysis、定位对比、市场竞争

**必填变量**：
- `[竞争对手名称]`：主要竞争对手

**选填变量**：
- `[分析重点]`：特定关注点（如：定价、产品、销售策略等）
- `[我们的优势]`：用于对比的自身优势

**提示词模板**：
```
准备一份针对[竞争对手名称]的深度竞争分析报告。

竞争对手：
[竞争对手名称]

分析重点（如有）：
[分析重点]

我们的优势（如有）：
[我们的优势]

分析要求：
请研究并撰写报告，包含以下内容：
1. 定价模式（价格区间、计费方式、折扣策略）
2. 产品定位（目标市场、核心功能、差异化点）
3. 近期客户流失情况（如可获得）
4. 销售策略和渠道（直销、渠道、合作伙伴等）
5. 与我们的优势对比分析
6. 他们的弱点和我们的机会

输出格式：
- 单页摘要（高管版）
- 详细分析报告（销售团队版）
- 附上引用和数据来源
```

**使用示例**：
- 用户："帮我分析一下Salesforce的定价和产品策略"
- Skill进行深度研究并生成对比报告

## 场景2：创建一份销售赋能单页文件

**触发关键词**：单页文件、one-pager、产品介绍、宣传资料、销售资料

**必填变量**：
- `[产品名称]`：要推介的产品或服务
- `[目标客户画像]`：理想客户的特征

**选填变量**：
- `[关键功能特性]`：核心功能列表
- `[竞争差异化]`：与竞品的主要区别
- `[客户案例]`：成功案例或数据

**提示词模板**：
```
制作一份单页宣传资料，帮助销售代表向[目标客户画像]推介[产品名称]。

产品信息：
- 产品名称：[产品名称]
- 目标客户：[目标客户画像]
- 关键功能：[关键功能特性]（如有）
- 差异化优势：[竞争差异化]（如有）
- 客户案例：[客户案例]（如有）

内容要求：
单页文件应包含：
1. 产品概述（1-2句话电梯演讲）
2. 关键优势（3-5个要点，每个1句话）
3. 核心功能特性（列表形式）
4. 常见应用场景（2-3个）
5. 与竞争对手的差异化优势（对比表格或要点）
6. 客户证明（案例、数据、引用）
7. 行动号召（联系方式、下一步）

格式要求：
- 结构清晰，易于扫读
- 可直接用于宣传的文档格式
- 视觉层次分明（标题、副标题、要点）
```

**使用示例**：
- 用户："做一个介绍我们AI客服产品的单页，给电商企业用"
- Skill生成结构化的单页宣传资料

## 场景3：在公共领域寻找客户证明点

**触发关键词**：客户评价、用户反馈、testimonial、社会证明、口碑调研

**必填变量**：
- `[产品或服务名称]`：要调研的产品（可以是自己的或竞品）

**选填变量**：
- `[调研重点]`：特定关注的方面（如：易用性、客服、性价比）
- `[对比维度]`：是否需要与竞品对比

**提示词模板**：
```
研究近期关于[产品或服务名称]的公开评价和用户反馈。

调研对象：
[产品或服务名称]

调研重点（如有）：
[调研重点]

对比需求（如有）：
[对比维度]

调研要求：
1. 搜索在线评论（G2、Capterra、TrustRadius等评测网站）
2. 社交媒体提及（Twitter、LinkedIn、Reddit等）
3. 用户评价和评分
4. 重点关注客户的赞扬和批评

输出内容：
1. 总结排名前5的正面评价（赞扬点）
2. 总结排名前5的负面评价（批评点）
3. 为每条评价注明：
   - 来源用户画像（行业、公司规模、职位）
   - 发布位置和链接
   - 评价日期
4. 整体趋势分析（情感倾向、核心主题）
5. 对销售的启示（如何利用正面评价、如何应对负面评价）

格式：结构化报告 + 引用链接
```

**使用示例**：
- 用户："帮我看看最近用户对我们产品的评价，尤其是和竞品HubSpot的对比"
- Skill进行公开信息调研并总结

## 使用说明

竞争情报场景帮助销售团队了解市场动态、竞争对手策略，以及如何有效定位和推介产品。

**关键特点**：
- 需要进行外部信息研究（WebSearch工具可能有用）
- 输出通常包含对比分析和战略建议
- 既服务于一线销售，也服务于销售管理层

**优先级**：
当用户提到"竞争对手"、"对比"、"市场分析"、"客户评价"等关键词时，优先匹配这些场景。

**注意事项**：
- 竞争分析应基于公开信息，避免涉及商业机密
- 客户证明点调研应注明来源，保证真实性
