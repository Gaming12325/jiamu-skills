# 对外联络与沟通场景库

本文件包含对外联络类别下的扩展场景（除SKILL.md中的2个高频场景外）。

---

## 场景1：为重要客户拟定续约方案

**触发关键词**：续约、合同续签、renewal、客户续约、续费方案

**必填变量**：
- `[客户名称]`：目标客户
- `[续约历史和价值数据]`：客户的历史使用数据、续约记录、价值评估

**选填变量**：
- `[投资回报率数据]`：量化的ROI证明
- `[续约建议]`：价格、服务等级等建议

**提示词模板**：
```
根据[客户名称]的续约历史和价值数据，撰写一份专业的续约方案。

客户数据：
[续约历史和价值数据]

ROI证明（如有）：
[投资回报率数据]

续约建议（如有）：
[续约建议]

输出要求：
1. 方案包含以下部分：
   - 客户价值回顾（过去一年的使用情况和成果）
   - 投资回报率证明（量化数据）
   - 续约建议（价格、服务等级、合同条款）
   - 额外价值提案（升级选项、新功能等）
2. 可选择附带一封后续邮件
3. 语气专业且以客户价值为导向

格式：结构化的方案文档 + 可选的邮件正文
```

**使用示例**：
- 用户："我要给ABC公司准备续约方案，他们去年用了我们的SaaS产品节省了30%成本"
- Skill识别为续约方案场景，收集客户数据后生成方案

## 场景2：创建代表活动摘要

**触发关键词**：活动摘要、工作总结、日报、周报、销售活动汇报

**必填变量**：
- `[销售代表活动数据]`：通话记录、会议记录、CRM活动导出等

**选填变量**：
- `[汇报周期]`：日报、周报、月报
- `[汇报对象]`：直接主管、团队、高管

**提示词模板**：
```
撰写销售代表的[汇报周期]工作总结，概述关键活动和成果。

活动数据：
[销售代表活动数据]

汇报对象：
[汇报对象]（如有）

输出要求：
1. 内容积极向上、简洁明了
2. 采用3-5条要点形式
3. 每条要点包含：活动类型、关键成果、后续计划
4. 突出业务进展和价值创造
5. 如果是向高管汇报，聚焦战略成果而非琐碎细节

输出格式：清晰的要点列表，适合口头汇报或书面提交
```

**使用示例**：
- 用户："帮我写今天的工作总结，我今天打了15个电话，开了3个会，成交了1单"
- Skill生成结构化的活动摘要

## 场景3：关于管道状态的执行官最新进展草案

**触发关键词**：高管汇报、管道汇报、项目储备汇报、pipeline update、执行官报告

**必填变量**：
- `[项目储备数据]`：当前销售管道的数据（总量、各阶段分布、关键交易等）

**选填变量**：
- `[汇报周期]`：本周、本月、本季度
- `[主要风险]`：已识别的风险因素
- `[最大成果]`：重大进展或成交

**提示词模板**：
```
向高管总结[汇报周期]我们的项目储备情况。

数据输入：
[项目储备数据]

主要风险（如有）：
[主要风险]

最大成果（如有）：
[最大成果]

输出要求：
以简短的高管报告形式撰写，包含：
1. 项目储备总量（总金额、交易数量）
2. 按销售阶段的分布情况
3. 主要风险和挑战
4. 最大成果和亮点
5. 预测信心度评估
6. 后续行动计划

格式：1页纸的高管摘要，数据可视化（如可能）+ 3-5句关键洞察
```

**使用示例**：
- 用户："我要向CEO汇报本月的销售管道，总共有500万在管道里，但有3个大单可能会延期"
- Skill生成结构化的高管汇报

---

## 使用说明

当用户的描述匹配到这些场景时，SKILL.md的智能适配规则会引用此文件中的相应模板。这些场景相对不那么高频，但在特定情况下非常有用。

**优先级**：
1. 先匹配SKILL.md中的6个高频场景
2. 如果不匹配，再检查此文件
3. 如果仍不匹配，继续检查其他reference文件

**扩展提示**：
如果发现某个场景使用频率上升，可以考虑将其提升为高频场景，加入SKILL.md主文件。
