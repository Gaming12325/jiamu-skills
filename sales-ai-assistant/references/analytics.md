# 数据分析与绩效洞察场景库

本文件包含数据分析类别下的扩展场景（除SKILL.md中的1个高频场景外）。

---

## 场景1：按成交率识别业绩最佳的销售代表

**触发关键词**：成交率、win rate、销售业绩、代表排名、业绩分析

**必填变量**：
- `[销售代表业绩数据]`：包含销售代表活动和已成交订单的数据集

**选填变量**：
- `[分析周期]`：季度、月度、年度
- `[细分维度]`：按产品线、地区、客户类型等细分

**提示词模板**：
```
根据以下销售代表业绩数据，计算每位代表的成交率并进行排名。

数据输入：
[销售代表业绩数据]
（建议包含：销售代表姓名、潜在客户数、已成交订单数、订单金额等）

分析周期（如有）：
[分析周期]

细分维度（如有）：
[细分维度]

分析要求：
1. 计算每位销售代表的成交率（已成交 / 潜在客户总数）
2. 按成交率从高到低排名
3. 计算平均成交率作为基准
4. 为每位代表提供描述性评价（如：表现优异、符合预期、需要改进）
5. 识别高绩效代表的共同特征（如可能）

输出格式：
- 排名列表（表格形式）
- 每位代表的优势描述
- 团队整体洞察
- 改进建议（针对低绩效代表）
```

**使用示例**：
- 用户："帮我分析一下团队的成交率，谁表现最好"
- Skill计算成交率并生成排名报告

## 场景2：可视化各季度的交易速度

**触发关键词**：交易速度、sales cycle、成交周期、deal velocity、销售周期

**必填变量**：
- `[交易数据]`：包含开盘日期和成交日期的CRM导出数据

**选填变量**：
- `[对比维度]`：按产品、地区、客户规模等对比
- `[目标周期]`：理想的成交周期时长

**提示词模板**：
```
使用以下CRM导出数据，计算每个季度的平均交易速度（从线索到成交的天数）。

数据输入：
[交易数据]
（需包含：交易ID、开盘日期、成交日期，可选：产品、地区、金额等）

对比维度（如有）：
[对比维度]

目标周期（如有）：
[目标周期]

分析要求：
1. 计算每笔交易的成交周期（成交日期 - 开盘日期）
2. 按季度汇总平均交易速度
3. 识别趋势（是否加快或变慢）
4. 如有对比维度，进行细分分析
5. 标注异常值（过快或过慢的交易）

输出格式：
- 简明图表形式展示速度趋势（文字描述图表）
- 趋势线总结（上升/下降/稳定）
- 关键洞察和建议
- 可能的影响因素分析
```

**使用示例**：
- 用户："我想看看过去一年每个季度的平均成交周期"
- Skill分析数据并可视化趋势

## 场景3：总结营销活动对已成交交易的归因

**触发关键词**：营销归因、campaign attribution、营销效果、活动ROI、来源分析

**必填变量**：
- `[营销活动和订单数据]`：营销活动来源与已成交订单的关联数据

**选填变量**：
- `[营销活动成本]`：各活动的投入成本
- `[归因模型]`：首次接触、最后接触、多点触达等

**提示词模板**：
```
根据以下数据，分析营销活动来源对已成交订单的贡献。

数据输入：
[营销活动和订单数据]
（需包含：订单ID、营销来源/活动名称、订单金额、成交日期等）

活动成本（如有）：
[营销活动成本]

归因模型（如有）：
[归因模型]

分析要求：
1. 将营销活动来源与已成交订单进行匹配
2. 确定哪个营销活动带来的成交收入最多
3. 计算每个活动的转化率和ROI（如有成本数据）
4. 识别高效和低效的营销渠道
5. 提供优化建议（增加投入或削减）

输出格式：
- 营销活动排名列表（按收入贡献）
- 每个活动的简短摘要
- ROI分析（如适用）
- 战略建议（资源重新分配）
```

**使用示例**：
- 用户："看看我们的营销活动哪个带来的订单最多"
- Skill进行归因分析并生成报告

## 场景4：生成性能对比图表

**触发关键词**：性能对比、业绩对比、performance comparison、销售对比分析

**必填变量**：
- `[对比数据]`：需要对比的业绩数据（按代表、地区、产品等）

**选填变量**：
- `[对比维度]`：具体对比什么（收入、成交数、增长率等）
- `[时间范围]`：对比的时间段

**提示词模板**：
```
以下是按[对比维度]的业绩数据，请进行对比分析。

对比数据：
[对比数据]

对比维度：
[对比维度]
（如：销售代表、产品线、地区、季度等）

时间范围（如有）：
[时间范围]

分析要求：
1. 比较业绩最佳和最差的对象
2. 绘制趋势图（文字描述图表形式）
3. 突出显示关键差异
4. 分析导致差异的可能原因
5. 提供改进建议

输出格式：
- 对比表格（清晰展示差异）
- 趋势图描述（如：柱状图、折线图等）
- 关键洞察分析
- 行动建议
```

**使用示例**：
- 用户："对比一下我们华东和华南两个区域的季度业绩"
- Skill生成对比分析和可视化描述

## 使用说明

数据分析场景帮助销售团队从数据中获取洞察，做出数据驱动的决策。

**关键特点**：
- 需要结构化数据输入（CSV、表格、CRM导出等）
- 输出包含定量分析和可视化建议
- 面向销售管理层和运营团队

**数据格式建议**：
用户可以：
- 直接粘贴CSV格式数据
- 上传文件（如支持）
- 用自然语言描述数据结构

**优先级**：
当用户提到"分析数据"、"计算"、"对比"、"趋势"等关键词，且提供了数据时，优先匹配这些场景。

**注意事项**：
- 提醒用户提供结构化数据以获得最佳结果
- 如果数据量大，建议先做数据清洗
- 保护客户数据隐私，不要在提示词中暴露敏感信息
